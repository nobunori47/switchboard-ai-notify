# Phase 2 提案: 電話自動応答 + テキスト化

提案先: 不動産管理会社様
提案日: 2026年7月

---

## 現状の課題

- 月50件程度の電話問い合わせのうち、営業時間外の取りこぼしが継続している
- 電話の内容がSlack・記録システムに乗らないため、対応履歴が断片化している
- メール・LINEはMVPで自動化されたが、電話だけ従来通りの手作業対応が残っている

---

## 提案内容

1. Twilioで050番号を取得し、着信を音声で受け付け
2. 音声をWhisper(またはClaude APIの音声対応機能)でテキスト化
3. テキスト化した内容を、既存のAI自動分類フロー(`inquiry_queue`)に合流させる
4. 結果として、メール・LINE・電話のすべてのチャネルが同じSlack通知フローに統合される

## アーキテクチャ(概要)

電話着信 → Twilio → 録音・文字起こし(Whisper) → inquiry_queue へ投入
↓
既存のClaude分類・Slack通知フローへ合流

MVPで構築した`inquiry_queue`テーブル・Claude分類ロジック・Slack通知ロジックは
そのまま再利用できるため、追加開発の範囲はWebhook受信部分(Twilio連携)に
限定されます。

---

## 費用・スケジュール

- 初期費用: 12万円(税込)
- 月額運用費: 既存の1〜2万円 + 5,000円程度(Twilio番号・通話料)
- 開発期間: 3営業日

---

## 期待される効果

- 電話・メール・LINEすべての問い合わせが一元管理される
- 営業時間外の電話取りこぼしを削減
- 対応履歴がすべて`inquiry_queue`に記録され、後から振り返り可能になる
